▶ "The data we work with is EEG."

4EEG Artefacts~2:30
▶ "And every recording is full of non-brain signal."
  • An artefact = any potential not from the brain; can be larger than the brain activity itself → dominates the trace (fig: blinks burying the frontal channels).
  • From the body (eyes, muscle, heart) and from outside (power line, movement).
  • They overlap the brain signal in time and frequency → can't just band-pass or threshold them away.
  • Slip through → corrupt the analysis (wrong artefacts in, wrong conclusions out).
Not here — the specific types (next slide).
5Artefact Types~3:15
▶ "Two families — and each behaves differently."
Physiological (from the body)
  • Ocular — eye movements/blinks; eyes sit next to the brain → most common & impactful; large, slow, low-frequency, frontal; measured by EOG.
  • Cardiac — the heartbeat; rhythmic sharp spikes mirroring the ECG, strongest over the left temporal area.
  • Muscle (EMG) — head/neck contractions; random, non-repetitive; below 1 Hz to over 200 Hz → hits every band; on all sensors → the hardest to remove.
Technical (external)
  • Line noise — 50 Hz mains, constant on all channels; removed by a notch filter.
  • Electrode/channel — poor contact/dry gel → jumps, flat lines, a single noisy electrode.
  • Movement — cable/electrode shifts → sudden deflections.   Environmental — nearby devices.
  • Punchline: line noise → preprocessing; physiological ones overlap the brain and must be separated → ICA. Muscle is the hard one.

6Independent Component Analysis~4:15
▶ "The standard way to separate them is ICA."
  • Unmixes the electrode mixture into statistically independent components, one per source.
  • Cocktail-party problem: microphones each pick up a mixture of voices; ICA recovers the voices. Electrodes = microphones.
  • X = A·S; estimate W ≈ A⁻¹ so Ŝ = W·X; works by maximising non-Gaussianity (FastICA).
  • Each component carries a topography, a spectrum, and a time course → these become the three views.
  • #components ≤ #electrodes; default 16 per recording.
Not here — classification (next slide).
7Manual Component Classification~5:00
▶ "But ICA doesn't tell you which component is brain and which is artefact."
  • An expert looks at every component and decides keep (brain) / reject (artefact) from the three properties.
  • Rejected ones removed by back-projection — signal rebuilt from kept components (fig: blinks red → gone in black).
  • Irreversible & asymmetric: wrong reject deletes real signal; wrong keep leaves noise → keep the human in control.
  • The bottleneck: hundreds of participants × ~16 components → slow, rushed, inconsistent; reasoning never recorded.

13Confidence Recommender~10:00
▶ "First building block — the recommender."
  • A keep-probability per component (1 = brain, 0 = artefact).
  • Three tiers (five bands, symmetric around 50%): Tier 1 ≥75% / ≤25% (batch-apply), Tier 2 60–75% / 25–40% (quick review), Tier 3 40–60% (manual).
  • Principle: classification with a reject option — uncertain cases go to the expert.
  • Muscle is hard → a separate badge from MNE's detector (faded 50–70%, solid >70%).
  • Indicator only — never pre-sets the decision; expert confirms each (came from the evaluation).

17Topomap~12:45
▶ "First detail view — topography, the expert's first criterion."
  • How strongly each sensor contributes + its polarity, from the ICA mixing matrix (µV); interpolated; blue → white → red.
  • The patterns classify: dipolar = brain · frontal = ocular · peripheral patch = muscle · single electrode = bad channel.
  • Hover = electrode name + amplitude; re-computes for a selected time range.

18Power Spectral Density~13:15
▶ "Second detail view — the spectrum, the other primary criterion."
  • Power across frequency, the five EEG bands shaded as background.
  • The shape names the source: falling 1/f (often an alpha peak) = brain · flat/rising = muscle · delta/theta = ocular.
  • Updates to whatever time range you select on the timeline.

  • Three overlays: EOG markers (blink events; if spikes align → ocular), trial markers (is the artefact even inside a trial?), PSD clusters.

20PSD Clustering~14:15
▶ "One overlay is worth a closer look — PSD clustering."
  • K-means on the spectra of overlapping time windows across the participant's components → colour-codes the timeline by spectral cluster.
  • Configurable cluster count, window size, step; similar spectral profiles get the same colour.
  • Payoff: see at a glance where a component's frequency character changes; hover a segment for the cluster's mean spectrum.
